Decision Variables (6 Total)

Pyomo model optimizes these variables at every timestep to minimize total system cost (LCOE).

Variable Type Bounds Description
TES_SOC[t] Continuous [0, 1,600 MWh] State of charge (thermal energy stored) at time t
TES_charge[t] Continuous [0, 300 MW] Charging power (electric → thermal) at time t
TES_discharge[t] Continuous [0, 100 MW] Discharging power (thermal input to turbine) at time t
turbine_on[t] Binary {0, 1} Turbine on/off status (enforces 40% minimum load constraint)
solar_to_TES[t] Continuous [0, solar[t]] Solar power allocated to TES charging (vs. direct datacenter use)
gas_backup[t] Continuous [0, 100 MW] Natural gas backup power (minimized by optimizer)
Binary Variables for Turbine On/Off: Physics constraint requires steam turbine to operate ≥40% thermal capacity. Binary variable turbine_on[t] enforces: If on: discharge ≥ 40 MW. If off: discharge = 0.

Optimization Constraints (8 Total)

Constraint Mathematical Formulation
1. Energy Balance SOC[t+1] = SOC[t] + charge[t] × η_heater × η_storage - discharge[t] / η_turbine
2. SOC Bounds 0 ≤ SOC[t] ≤ 1,600 MWh (storage capacity limit)
3. Charge Power Limit 0 ≤ charge[t] ≤ 300 MW (heater capacity limit)
4. Discharge Power Limit 0 ≤ discharge[t] ≤ 100 MW (turbine thermal capacity)
5. Minimum Load (40%) discharge[t] ≥ 40 MW × turbine_on[t] AND discharge[t] ≤ 100 MW × turbine_on[t]
6. Power Balance solar[t] + wind[t] + TES_electric[t] + gas[t] = 100 MW (datacenter load)
7. Solar Allocation solar_direct[t] + solar_to_TES[t] ≤ solar[t]
8. Gas Limit (CFE Target) Σ gas[t] ≤ target_gas_MWh (e.g., 0 for 100% CFE)
MINIMUM LOAD (40%) IMPLEMENTATION: Uses "Big-M" method with binary variable. If turbine_on[t] = 1: 40 MW ≤ discharge[t] ≤ 100 MW. If turbine_on[t] = 0: discharge[t] = 0. Prevents unrealistic partial loads.

Steam Turbine Efficiency Curve Implementation

Efficiency varies with load factor (40% to 100%). Pyomo model uses linear interpolation between validated bounds.

def calculate_turbine_efficiency(discharge_MW, thermal_capacity_MW=100,
                                   eff_at_40pct=0.34, eff_at_100pct=0.39):
    """
    Calculate steam turbine efficiency based on load factor.
    
    Validated parameters (Phase 1):
    - eff_at_40pct = 0.34 (EPA CHP Catalog + NREL SAM)
    - eff_at_100pct = 0.39 (EPA CHP Catalog, 36-40% range)
    
    Linear interpolation between 40% and 100% load.
    """
    load_factor = discharge_MW / thermal_capacity_MW
    
    if load_factor < 0.4:
        return 0  # Turbine off (below minimum load)
    
    # Linear interpolation
    efficiency = eff_at_40pct + (load_factor - 0.4) / 0.6 * (eff_at_100pct - eff_at_40pct)
    
    return efficiency

Rolling 48-Hour Optimization Windows

WHY 48-HOUR WINDOWS? More efficient AND better represents reality than full-year optimization. Captures multi-day weather patterns while giving LDES incentive to bank energy.

Implementation Logic

  • Weather variability: Solar/wind availability spans multiple days. Single-day optimization can't anticipate tomorrow's generation.
  • TES duration: 16-hour storage means decisions today affect tomorrow's backup availability. 48-hour window provides sufficient lookahead.
  • Rolling window rationale: More efficient than full-year MIP and better represents reality. Tuning inputs give LDES incentive to bank energy.
  • 24-hour step: Window advances 24 hours each iteration, implementing only first 24 hours of optimized schedule.
for day in range(0, 365, 1):  # Daily steps
    window_start = day * 24
    window_end = window_start + 48  # 48-hour window
    
    # Extract data for 48-hour window
    solar_window = solar_profile[window_start:window_end]
    wind_window = wind_profile[window_start:window_end]
    
    # Solve optimization for 48-hour window
    results = solve_pyomo_model(solar_window, wind_window, initial_SOC)
    
    # Implement ONLY first 24 hours of solution
    implemented_dispatch = results.dispatch[:24]
    
    # Update initial conditions for next window
    initial_SOC = results.SOC[24]

Performance: Each 48-hour window solves in ~2 seconds. Efficient and scalable for daily operations.

LCOE Calculator: IRA 30% Investment Tax Credit

Python module includes LCOE calculator incorporating IRA Section 48E (30% ITC for energy storage).

LCOE = (Annual_Capital_Cost + Annual_OpEx) / Annual_Generation_MWh

Where:
  Annual_Capital_Cost = Net_CapEx × CRF(WACC, lifetime)
  Net_CapEx = Gross_CapEx × (1 - IRA_ITC_rate)
  IRA_ITC_rate = 0.30 (30% credit through 2032)
  CRF = Capital Recovery Factor = WACC × (1+WACC)^n / ((1+WACC)^n - 1)
